# 海外周报自动生成流程

## 触发方式

用户说"生成周报"、"跑一下周报"时触发。

## 流程步骤

### 1. 获取周报文件夹全部文件

- 文件夹 token: `YOUR_FOLDER_TOKEN`
- 调用 `feishu_drive_file` list 该文件夹
- 筛选 type=file 的文件（排除文件夹）

### 2. 从文件名解析周报信息

解析规则：
- **日期提取**：从文件名中提取数字串，优先匹配 `yyyyMMdd` 格式（如 `20260403` → 2026年4月3日）
- **名称提取**：去掉日期和扩展名后的部分（如 `亚洲烯烃周报20260403.pdf` → 亚洲烯烃周报）
- 常见文件名模式示例：
  - `Olefinscan_02日 4月 2026年.pdf` → 名称: Olefinscan，日期: 2026年4月2日
  - `阿格斯乙烯周报20260402.pdf` → 名称: 阿格斯乙烯周报，日期: 2026年4月2日
  - `亚洲烯烃周报20260403.pdf` → 名称: 亚洲烯烃周报，日期: 2026年4月3日

### 3. 下载 PDF

- 调用 `feishu_drive_file` download，保存到 `/tmp/openclaw/weekly_report_latest.pdf`

### 4. 按 Playbook 提示词分析 PDF

- 读取 `playbooks/海外周报总结-核心指令.md` 作为角色和约束指令
- 读取 `playbooks/海外周报总结-输出格式模板.md` 作为格式指令
- 调用 `pdf` 工具，将提示词 + PDF 一起传入
- 要求输出完整的七段式分析报告

### 5. 生成飞书文档

- 调用 `feishu_create_doc`
- **标题**：`[解析出的名称] 市场分析报告（解析出的日期）`，例如 `亚洲烯烃周报 市场分析报告（2026年4月3日）`
- **存储位置**：指定 `folder_token: YOUR_FOLDER_TOKEN`，直接存到周报文件夹
- **末尾署名**：`设计:Orange | 生成:OpenClaw`
- 返回文档链接给用户

## 异常处理

| 异常场景 | 处理方式 |
|---------|---------|
| 文件夹为空或无PDF | 提示用户"周报文件夹暂无文件，请先上传周报" |
| 下载失败（权限不足） | 引导用户完成授权后自动重试 |
| PDF 损坏/无法解析 | 提示用户"PDF无法解析，请确认文件完整"，跳过该文件 |
| 文件名无法解析日期 | 使用当前日期作为默认值，并在标题标注"（日期待确认）" |
| 模型输出格式不完整 | 检查是否缺少必填章节，如有缺失则提示用户"报告可能不完整，建议检查" |
| 飞书文档创建失败 | 提示用户"文档创建失败，请稍后重试"，将内容以消息形式直接发送给用户作为备选 |
| API 超时/网络异常 | 等待10秒后重试一次，仍失败则提示用户 |
