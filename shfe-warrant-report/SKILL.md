## 流程

**全程不要输出中间状态文字，只在最后一次性输出结果。**

### 1. 打开页面并筛选品种

浏览器打开上期所仓单日报页面：
```
https://www.shfe.com.cn/reports/tradedata/dailyandweeklydata/?query_params=dailystock
```

**⚠️ 绝对禁止直接全页 snapshot！必须先筛选品种。**

页面加载后，点击右侧侧边栏的目标品种链接（如「丁二烯橡胶」），页面会滚动到对应板块。

### 2. 获取数据

等待页面稳定后，使用 `browser act` + `evaluate` 从 DOM 提取结构化数据。**禁止使用 snapshot 文本手动提取，必须用 JS 精确提取。**

```javascript
// 点击侧边栏品种链接后，执行以下 JS 提取
() => {
  // 找到目标品种的表格区域
  const body = document.body.innerText;
  // 用品种名定位数据段
  // 提取仓库和厂库板块的所有行数据
  const tables = document.querySelectorAll('table');
  // ...按品种筛选后返回 JSON
}
```

**更可靠的方案**：直接用 `evaluate` 搜索页面文本，提取该品种（仓库+厂库）的完整表格数据为 JSON 数组。每个条目包含：`{type: '仓库'|'厂库', region, name, amount, change}`。

如果 DOM 提取失败（如表格结构异常），退而用 `snapshot compact=true maxChars=20000`，但**必须逐行核对**确保不遗漏任何增减量≠0的仓库。

### 3. 输出结果

通过飞书消息一次性发送：

**仓库变化量明细：**
仅列出增减量 ≠ 0 的仓库，格式：仓库名 变化量（如"中储临港 -170"），按地区分组，含地区小计。**必须列出所有地区，不允许遗漏。**

**厂库变化量明细：**
仅列出增减量 ≠ 0 的厂库，格式同上。如无厂库数据则省略。**必须列出所有地区，不允许遗漏。**

**汇总：**
- 仓单总量（仓库 + 厂库）
- 当日总变化量

## 注意事项

- **禁止全页 snapshot**，必须先通过侧边栏点击筛选到目标品种，再 snapshot
- 如果侧边栏品种链接不可点击，备选方案：用 `browser act` 点击筛选栏中的品种文字
- 如果用户未指定品种，询问需要哪个品种
- 如果用户说"全部"，则逐个品种点击筛选、snapshot、整理
- 禁止使用模拟数据
